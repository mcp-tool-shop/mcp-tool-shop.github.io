Context Window Manager

Lossless context restoration via KV cache persistence. Freeze, thaw, and clone contexts.

366 tests passing | Beta v0.6.2 | Production hardening complete
pip install context-window-manager

Requires vLLM with prefix caching and LMCache

Traditional (Lossy)

Context fills up → Summarize → Lose details. Every compaction loses nuance. Re-explain constraints. Re-make decisions.

CWM (Lossless)

Context fills up → Freeze KV cache → Store tensors → Thaw later. Exact restoration, zero loss.

Freeze Sessions

Snapshot your current context to persistent storage. KV cache tensors preserved exactly.

Thaw Anytime

Restore context later with zero information loss. Resume exactly where you left off.

Clone Contexts

Branch a context to explore different conversation paths. Perfect for experimentation.

Tiered Storage

Automatic management: GPU → CPU → Disk → Redis. Optimal cost-performance.

Session Isolation

Unique cache_salt per session. No cross-session data leakage. Timing attack protection.

Auto-Freeze

Configure automatic freezing at token thresholds. Never lose context unexpectedly.

Usage

# Freeze your current session
window_freeze session_abc123 my-coding-project

# Later, restore it
window_thaw my-coding-project

# Clone for experimentation
window_clone my-coding-project experiment-branch

# List all saved windows
window_list

# Check detailed status
window_status my-coding-project
