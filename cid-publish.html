Publisher Quickstart

Publish a CID-verified tool in 3 steps

1 Create the tool bundle

Package your tool files and compute the manifest CID.

# Install the packer (reference implementation) npm install -g @mcp-tool-shop/packer # Pack your tool directory mcp-pack ./my-tool --output my-tool.car # Output: # root_cid: bafyrei... # descriptor_cid: bafyrei... # manifest: my-tool.manifest.json # descriptor: my-tool.descriptor.json
The manifest CID is computed over {schema_version, cid_profile, entries} only. See Rulebook §4.2.3.

2 Produce an attestation

Sign an integrity claim with your Ed25519 key.

# Generate a key pair (if you don't have one) mcp-keygen --output my-key.json # Create and sign an attestation mcp-attest \ --root-cid bafyrei... \ --descriptor-cid bafyrei... \ --key my-key.json \ --role maintainer \ --output attestation.json
Attestations use the signing preimage format from Rulebook §3. The signed_fields array specifies which JSON Pointers are covered.

3 Verify locally

Run the verifier to confirm everything passes.

# Clone the reference verifier git clone https://github.com/mcp-tool-shop/mcp-registry-spec.git cd mcp-registry-spec/verifier npm install && npm run build # Verify your bundle npm run verify -- \ --pointer pointer.json \ --descriptor my-tool.descriptor.json \ --manifest my-tool.manifest.json \ --attestation attestation.json # Expected output: # result: ACCEPT # provenance: # tool: my-tool # root_cid: bafyrei...

Run conformance vectors

Verify your implementation against the official test suite.

cd mcp-registry-spec/verifier npm test # Expected: # tests 24 # pass 24 # fail 0
